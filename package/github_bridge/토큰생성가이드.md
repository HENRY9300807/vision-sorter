# GitHub 토큰 생성 가이드

## 빠른 링크

- **Fine-grained token (권장)**: https://github.com/settings/tokens/new?type=beta
- **Classic token**: https://github.com/settings/tokens/new

## Fine-grained Token 생성 방법

1. 위 링크로 이동 (또는 GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens)

2. **"Generate new token"** 버튼 클릭 (오른쪽 상단)

3. 설정 입력:
   - **Token name**: `GitHub Bridge`
   - **Expiration**: 원하는 기간 선택 (예: 90 days)
   - **Description**: `GitHub Bridge API access` (선택)

4. **Repository access**:
   - `Selected repositories` 선택
   - `vision-sorter` 레포지토리 선택

5. **Repository permissions** 설정:
   - **Contents**: `Read` ✅
   - **Metadata**: `Read` ✅
   - **Issues**: `Read` (선택)
   - **Pull requests**: `Read` (선택)
   - **Commit statuses**: `Read` (선택)

6. **"Generate token"** 버튼 클릭 (하단)

7. 생성된 토큰 복사 (⚠️ 한 번만 표시됨!)

## Classic Token 생성 방법 (대안)

1. https://github.com/settings/tokens/new 접속

2. **"Generate new token (classic)"** 클릭

3. 설정:
   - **Note**: `GitHub Bridge`
   - **Expiration**: 원하는 기간
   - **Select scopes**:
     - ✅ `repo` (전체 레포지토리 접근)
     - 또는 필요한 권한만 선택

4. **"Generate token"** 클릭

5. 토큰 복사

## 토큰 설정

토큰을 생성한 후 PowerShell에서:

```powershell
# 방법 1: 스크립트 사용
.\package\github_bridge\set_token.ps1 -Token 'ghp_여기에_토큰'

# 방법 2: 직접 설정
$env:GITHUB_TOKEN = 'ghp_여기에_토큰'
.\package\github_bridge\test_api.ps1
```
