# File Path 입력 가이드

## File Path란?

GitHub 레포지토리에서 가져올 **파일의 경로**를 입력하는 필드입니다.

## 경로 형식

레포지토리 루트 기준으로 파일 경로를 입력합니다.

### 예시

레포지토리 구조:
```
vision-sorter/
├── main.py
├── package/
│   ├── capture_96_limit.py
│   ├── image_utils.py
│   └── operation.py
├── ui/
│   └── color_definition.py
└── README.md
```

### 올바른 입력 예시

| 파일 | File Path 입력 |
|------|---------------|
| 루트의 main.py | `main.py` |
| package/capture_96_limit.py | `package/capture_96_limit.py` |
| package/image_utils.py | `package/image_utils.py` |
| ui/color_definition.py | `ui/color_definition.py` |
| README.md | `README.md` |

### 잘못된 입력 예시

❌ `/main.py` (앞에 슬래시 붙이지 않기)
❌ `C:\Users\...\main.py` (로컬 경로 아님)
❌ `./main.py` (상대 경로 표시 불필요)

## 여러 파일 추가하기

1. 첫 번째 파일 경로 입력 → **Add** 클릭
2. 두 번째 파일 경로 입력 → **Add** 클릭
3. 반복...

예시:
- `main.py` 입력 → Add
- `package/capture_96_limit.py` 입력 → Add
- `package/image_utils.py` 입력 → Add

## 디렉토리 구조 확인하기

레포지토리의 파일 구조를 확인하려면:

1. GitHub 웹사이트에서 레포지토리 열기
2. Code 탭에서 파일 목록 확인
3. 파일을 클릭하면 URL에 경로가 표시됨
   - 예: `https://github.com/HENRY9300807/vision-sorter/blob/main/package/capture_96_limit.py`
   - 경로: `package/capture_96_limit.py`

## 팁

- **파일명만 입력**: 루트에 있는 파일은 파일명만 입력
- **슬래시 사용**: 디렉토리 구분은 `/` 사용 (Windows든 Linux든)
- **대소문자 구분**: GitHub는 대소문자를 구분합니다
- **확장자 포함**: `.py`, `.txt` 등 확장자까지 정확히 입력

## 문제 해결

### 파일을 찾을 수 없다는 오류가 나면?

1. GitHub에서 파일 경로 확인
2. 대소문자 확인
3. 슬래시 방향 확인 (`/` 사용)
4. 파일이 실제로 존재하는지 확인

### 디렉토리 내용을 보고 싶다면?

현재 GUI는 개별 파일만 지원합니다. 디렉토리 내용을 보려면:
- GitHub 웹사이트에서 직접 확인
- 또는 API를 직접 호출
